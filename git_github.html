Git & GitHub (Full Course)

"코드 날려먹었어요"라는 말은 이제 그만. 코딩계의 타임머신이자 개발자들의 필수 이력서인 Git과 GitHub의 기초부터 협업 워크플로우까지 완벽하게 박살내 드립니다.

1. Git과 GitHub, 같은 거 아냐?

이름은 비슷하지만 완전히 다른 역할을 합니다. 이 둘의 차이를 아는 것이 출발점입니다.

  • Git (깃): 내 컴퓨터(로컬) 안에서 코드의 변경 내역을 찰칵찰칵 사진 찍듯 저장해주는 버전 관리 프로그램(타임머신)입니다. 인터넷이 없어도 작동합니다.
  • GitHub (깃허브): 내 컴퓨터에 저장해둔 Git 기록들을 클라우드(인터넷)에 올려 보관하고, 남들과 공유할 수 있게 해주는 웹 서비스(구글 드라이브 같은 곳)입니다.
💡 초기 필수 세팅 (최초 1회만) Git을 처음 설치했다면, 누가 코드를 수정했는지 기록하기 위해 이름과 이메일을 등록해야 합니다.
터미널을 열고 다음을 입력하세요:
git config --global user.name "내이름"
git config --global user.email "내이메일@example.com"

2. 로컬에서 작업하기 (박스 포장 3단계)

코드를 수정하고 저장(Commit)하는 과정은 택배 박스를 포장하는 것과 완벽히 똑같습니다.

init → add → commit

폴더를 저장소로 만들고(init), 보낼 물건만 박스에 담고(add), 박스를 테이프로 밀봉하며 라벨을 붙이는(commit) 과정입니다.

# 1. 이 폴더를 Git이 감시하도록 시작 (저장소 생성) git init # (코드를 열심히 수정합니다...) # 2. 어떤 파일들이 수정되었는지 상태 확인 git status # 3. 변경된 모든 파일( . )을 포장 박스(Staging Area)에 담기 git add . # 4. 박스를 밀봉하고 메모(Commit Message) 남기기 git commit -m "메인 페이지 로그인 버튼 추가"
> Terminal - bash
Initialized empty Git repository in /project/.git/

[master (root-commit) a1b2c3d] 메인 페이지 로그인 버튼 추가
1 file changed, 15 insertions(+)
create mode 100644 index.html

3. GitHub로 쏘아 올리기 (Push)

내 컴퓨터에 포장된 박스(Commit)들이 쌓였다면, 이제 GitHub(원격 저장소)라는 창고로 배송(Push)할 차례입니다.

# 1. 내 컴퓨터 폴더와 GitHub 주소 연결하기 # (origin이라는 별명으로 이 주소를 기억해라!) git remote add origin https://github.com/아이디/레포지토리.git # 2. 내 컴퓨터의 main 가지(브랜치)에 있는 박스들을 # origin(GitHub)으로 업로드하기 (-u는 처음 연결 시 사용) git push -u origin main
> Terminal - bash
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Writing objects: 100% (3/3), 302 bytes | 302.00 KiB/s, done.
To https://github.com/아이디/레포지토리.git
* [new branch] main -> main
💡 팀원이 수정한 걸 내 컴퓨터로 가져오려면? 다른 팀원이 GitHub에 새 코드를 올렸다면, 내가 작업하기 전에 먼저 내려받아야 충돌이 안 납니다. 이땐 git pull origin main 을 입력하세요!

4. 평행 우주 만들기: 브랜치 (Branch)

메인 코드를 건드리면 서비스가 고장 날까 봐 두렵나요? 브랜치를 따면 기존 코드는 안전하게 둔 채로, 나만의 복사본(평행 우주)에서 마음껏 실험하고 개발할 수 있습니다.

# 1. 'login-feature' 라는 새로운 가지(평행우주) 만들기 git branch login-feature # 2. 내가 작업할 공간을 새 가지로 이동(Switch)하기 # (이제부터 여기서 코드를 망쳐도 main은 안전합니다) git switch login-feature # --- 로그인 기능 개발 완료! (add & commit 완료 가정) --- # 3. 완벽하게 개발된 기능을 원본(main)에 합치기 위해 main으로 돌아감 git switch main # 4. 내 평행우주(login-feature)의 코드를 main으로 병합(Merge)해라! git merge login-feature

5. 이거 올리지 마! .gitignore

Git이 모든 파일을 다 백업할 필요는 없습니다. 비밀번호가 담긴 파일, node_modules처럼 용량이 너무 큰 라이브러리 폴더 등은 GitHub에 올라가면 큰일 납니다.

# 프로젝트 최상단에 .gitignore 라는 파일을 만들고 그 안에 적어주세요. # 폴더 전체를 무시 node_modules/ build/ # 특정 확장자 파일 다 무시 *.log *.env # API 키, 비밀번호 등이 들어있는 보안 파일 # 특정 파일 딱 1개 지정해서 무시 secret_password.txt
⚠️ 이미 올려버렸다면? 만약 .gitignore에 적기 전에 이미 git add나 commit을 해버렸다면, 나중에 무시 목록에 적어도 계속 추적됩니다. 꼭 프로젝트 시작 시 제일 먼저 .gitignore를 만들어주세요!
