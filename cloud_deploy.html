클라우드 & 배포 (Deploy) 완벽 가이드

localhost:3000 지옥에서 탈출할 시간입니다. 친구에게 스마트폰으로 접속할 수 있는 '진짜 주소'를 만들어주는 클라우드 서버와 배포의 마법을 배워봅시다.

1. 배포(Deployment)가 대체 뭔가요?

우리가 코딩할 때 보는 http://localhost:3000은 '내 컴퓨터(로컬)' 안에서만 돌아가는 가짜 인터넷입니다. 이 코드를 24시간 내내 켜져 있고, 전 세계 어디서든 접속할 수 있는 남의 컴퓨터(서버)에 올려두는 과정을 **배포**라고 합니다.

  • 호스팅(Hosting): 남의 컴퓨터(서버)의 공간을 월세 내고 빌리는 것 (예: AWS, GCP)
  • 클라우드(Cloud): 물리적인 서버 컴퓨터를 직접 사지 않고, 인터넷을 통해 가상으로 서버를 빌려 쓰는 현대적인 방식

2. 프론트엔드 1분 배포 (Vercel / Netlify)

HTML, CSS, React로 만든 프론트엔드 화면은 서버 세팅을 할 필요가 없습니다. GitHub 레포지토리를 연결만 하면 버튼 한 번에 전 세계로 배포해 주는 미친 서비스들이 있습니다.

Vercel (버셀) - React/Next.js의 신

GitHub에 내 코드를 push 하기만 하면, Vercel이 그걸 감지해서 자동으로 빌드(Build)하고 나만의 도메인(링크)을 만들어줍니다. 심지어 무료입니다!

# 터미널에서 명령어로도 바로 배포할 수 있습니다. # (프로젝트 폴더 안에서) # 1. vercel 프로그램 설치 npm i -g vercel # 2. 내 코드 Vercel 클라우드로 쏘아올리기 vercel # (몇 가지 질문에 Y/Enter 누르면 끝!)

3. 백엔드 서버 (AWS EC2 vs PaaS)

데이터베이스를 다루고 로직을 계산하는 백엔드(Java, Node.js, Python)는 프론트엔드보다 복잡합니다. 리눅스 운영체제가 깔린 '진짜 컴퓨터'를 빌려야 합니다.

초보자를 위한 추천 테크트리

  • 1단계 (PaaS): Render, Heroku - 초보자용. 코드만 주면 자기들이 알아서 컴퓨터 세팅하고 실행해 줍니다. 매우 편하지만 나중에 요금이 비싸질 수 있습니다.
  • 2단계 (IaaS): AWS EC2 - 전문가용. 빈 깡통 컴퓨터(주로 리눅스)만 달랑 빌려줍니다. 우리가 직접 접속해서 Java 깔고, DB 깔고, 포트 열어줘야 하지만, 실무에서는 99% 이걸 씁니다.
💡 서버비 폭탄 주의! AWS 가입할 때 카드 번호를 넣게 되는데, 무료로 제공하는 '프리티어(Free Tier)' 서버 사양을 넘어서거나 코딩 실수로 무한 루프가 돌면 한 달에 수십만 원이 청구될 수 있습니다. 꼭 요금 알림(Billing Alarm)을 설정하세요!

4. 도커 (Docker) - "내 컴에선 되는데?" 방지용

로컬(내 맥북)에서는 쌩쌩 돌아가던 서버가, AWS 리눅스 컴퓨터에 올렸더니 버전이 달라서 에러를 뿜는 경우가 엄청 많습니다. 이를 해결하기 위한 기술이 컨테이너(Container)입니다.

도커의 마법: 코드와 환경을 박스 하나에!

내 코드뿐만 아니라, 그 코드가 돌아가는 데 필요한 운영체제, Java 버전, 환경설정까지 전부 하나의 '컨테이너 박스'에 담아서 묶어버립니다. 이 박스는 윈도우든 맥이든 리눅스든 어디서나 똑같이 켜집니다.

# Dockerfile (도커 박스를 만드는 레시피) # 1. 뼈대: Node.js 18버전이 깔린 리눅스 환경을 가져와라 FROM node:18-alpine # 2. 작업 폴더 지정 WORKDIR /app # 3. 내 컴퓨터에 있는 패키지 설정 파일 복사 COPY package.json . # 4. 박스 안에서 라이브러리 설치 RUN npm install # 5. 내 소스코드들 전부 박스 안으로 복사 COPY . . # 6. 박스 켜질 때 실행할 명령어 CMD ["npm", "start"]

5. 도메인(Domain)과 HTTPS (보안)

서버를 열면 123.45.67.89:8080 같은 흉측한 IP 주소가 생깁니다. 여기에 예쁜 이름을 붙여주고, 자물쇠 마크(보안)를 다는 과정입니다.

  • 도메인 (Domain): 가비아, 호스팅케이알 같은 곳에서 1년에 1~2만 원 주고 my-project.com 같은 이름표를 사는 겁니다.
  • DNS (Domain Name System): 사람들이 my-project.com을 쳤을 때, 내 AWS 서버의 숫자 IP 주소(123.45.67.89)로 길을 안내해 주는 전화번호부 역할입니다.
  • HTTPS (SSL 인증서): 주소창 옆에 뜨는 '자물쇠' 마크입니다. 통신을 암호화해 주며, 요즘 이거 없으면 크롬 브라우저가 접속을 막아버립니다.
