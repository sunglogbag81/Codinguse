Chapter 1. 웹 프론트엔드 시작

HTML 뼈대 세우기

어렵고 복잡한 연산은 필요 없습니다. 우리가 매일 보는 웹사이트의 구조를 텍스트로 블록 조립하듯 만들어보는 첫걸음입니다.

1. 태그(Tag)란 무엇인가?

HTML의 전부라고 해도 과언이 아닙니다. 브라우저에게 "이 텍스트는 제목이야", "이건 버튼이야"라고 알려주기 위해 이름표(태그)를 붙여주는 것이죠.

기본 문법: 열었으면 닫아라!

대부분의 태그는 한 쌍으로 움직입니다. <태그명>으로 열고, </태그명>으로 꼭 닫아주어야 합니다.

<h1>가장 큰 제목</h1> <h3>중간 제목</h3> <p>일반적인 문단입니다. <br> br 태그는 줄바꿈을 해줍니다.</p>

가장 큰 제목

중간 제목

일반적인 문단입니다.
br 태그는 줄바꿈을 해줍니다.

2. HTML 문서의 기본 뼈대

사람에게 머리와 몸통이 있듯, HTML도 눈에 보이지 않는 설정을 담당하는 head와 실제 내용이 들어가는 body로 나뉩니다. VS Code에서 ! 치고 엔터를 누르면 자동 완성됩니다.

<!-- 문서 버전 명시 --> <!DOCTYPE html> <html lang="ko"> <head> <!-- 눈에 보이지 않는 설정 --> <meta charset="UTF-8"> <title>브라우저 탭 이름</title> </head> <body> <!-- 우리가 화면에서 보는 모든 것 --> </body> </html>

3. 링크와 이미지 (속성)

"어디로 갈지", "어떤 사진을 띄울지" 등 추가 정보를 태그 안에 적어주는 것을 속성(Attribute)이라고 합니다.

a 태그와 img 태그

a(Anchor) 태그는 href 속성을, img 태그는 src 속성을 사용합니다.

<a href="https://google.com">구글 가기</a> <br><br> <!-- 닫는 태그 없음 --> <img src="photo.jpg" alt="설명" width="100">
구글 가기
(이미지)

4. 목록과 표 만들기

정보를 나열할 때는 리스트(List)를, 격자 형태로 보여줄 때는 테이블(Table)을 사용합니다.

ul, ol, li 태그

ul은 순서 없는 목록(점), ol은 순서 있는 목록(숫자)입니다. 항목은 li로 적습니다.

<ul> <li>사과</li> <li>바나나</li> </ul> <ol> <li>첫 번째</li> <li>두 번째</li> </ol>
  • 사과
  • 바나나
  1. 첫 번째
  2. 두 번째

table 태그

tr은 행(가로줄), th는 제목 셀, td는 일반 데이터 셀입니다.

<table border="1"> <tr> <th>이름</th> <th>나이</th> </tr> <tr> <td>홍길동</td> <td>20</td> </tr> </table>
이름 나이
홍길동 20

5. 입력창 (Form & Input)

로그인이나 회원가입처럼 사용자의 입력을 받을 때 사용합니다.

다양한 input 타입

type 속성을 바꾸면 텍스트, 비밀번호, 체크박스 등 모양이 변합니다.

<input type="text" placeholder="아이디"><br> <input type="password" placeholder="비밀번호"><br> <input type="checkbox"> 동의합니다<br> <button>제출</button>



6. 구역 나누기 & 시맨틱 태그

웹페이지를 디자인하기 위해 투명한 상자로 레이아웃을 나눕니다.

div 와 span

div는 한 줄을 다 차지하는 큰 박스, span은 글자 크기만큼만 차지하는 작은 박스입니다.

<div style="background: yellow;">박스1</div> <div style="background: orange;">박스2</div> <span style="background: lightblue;">글자1</span> <span style="background: lightgreen;">글자2</span>
박스1
박스2
글자1 글자2
💡 시맨틱(Semantic) 태그란? 의미 없는 div 대신, 역할을 명확히 알 수 있게 만든 태그입니다. (예: <header>, <nav>, <main>, <footer>) 겉모양은 div와 똑같지만 검색 엔진이 웹사이트를 잘 이해하도록 돕습니다.
