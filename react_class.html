React 핵심 개념 (Full Course)

수많은 웹페이지를 복붙해서 만들던 시대는 끝났습니다. UI를 레고 블록처럼 조립하고, 데이터가 바뀌면 화면이 알아서 변하는 리액트의 세계에 오신 것을 환영합니다.

1. 컴포넌트 (Components)

리액트의 가장 기본 단위입니다. 긴 HTML 코드를 하나의 함수로 묶어서, 마치 나만의 맞춤형 태그를 만드는 것과 같습니다.

// 나만의 Header 컴포넌트 만들기 function Header() { return ( <header> <h1>안녕하세요, 리액트!</h1> </header> ); } // 레고 블록 조립하듯 재사용하기 function App() { return ( <div> <Header /> <Header /> </div> ); }
<Header />

안녕하세요, 리액트!

<Header />

안녕하세요, 리액트!

💡 컴포넌트 이름 규칙 리액트에서 컴포넌트의 이름은 반드시 대문자로 시작해야 합니다. 소문자로 시작하면 리액트가 일반 HTML 태그(예: div, header)로 오해합니다.

2. JSX (JavaScript XML)

자바스크립트 파일 안에서 HTML을 쓸 수 있게 해주는 마법의 문법입니다. HTML과 거의 똑같지만, 몇 가지 중요한 차이점이 있습니다.

JSX의 3가지 규칙

1) class 대신 className을 씁니다.
2) 자바스크립트 변수를 쓸 때는 중괄호 { }로 감쌉니다.
3) 모든 태그는 반드시 닫혀야 합니다. (예: <br />, <img />)

function Profile() { // 자바스크립트 변수 const name = "개발자"; const color = "blue"; return ( <div className="box"> {/* 중괄호 안에서 JS 변수 사용! */} <h2>내 이름은 {name}</h2> {/* 스타일도 중괄호 2개로 객체 형태로! */} <p style={{ color: color }}> 파란색 글씨입니다. </p> </div> ); }

내 이름은 개발자

파란색 글씨입니다.

3. Props (데이터 물려주기)

부모 컴포넌트가 자식 컴포넌트에게 데이터를 전달할 때 사용하는 방법입니다. HTML의 속성(Attribute)을 적는 것과 똑같은 방식입니다.

// 자식: props를 받아서 화면에 그림 function UserCard(props) { return ( <div className="card"> <h3>이름: {props.name}</h3> <p>나이: {props.age}</p> </div> ); } // 부모: 자식에게 이름과 나이를 전달 function App() { return ( <div> <UserCard name="홍길동" age={20} /> <UserCard name="김코딩" age={25} /> </div> ); }
UserCard (name="홍길동", age=20)

이름: 홍길동

나이: 20

UserCard (name="김코딩", age=25)

이름: 김코딩

나이: 25

4. State (상태 관리) ★가장 중요

일반 JS 변수는 값이 바뀌어도 화면이 그대로입니다. 하지만 리액트의 State는 값이 바뀌면 화면을 알아서 다시 새로고침(Re-render) 해주는 마법의 데이터입니다.

useState 훅(Hook) 사용하기

[현재값, 값을바꿀함수] = useState(초기값) 형태로 사용합니다.

import { useState } from 'react'; function Counter() { // count는 0으로 시작, setCount로 값을 바꿈 const [count, setCount] = useState(0); return ( <div> <p>현재 카운트: {count}</p> {/* 버튼을 누르면 setCount로 count를 1 증가시킴 */} <button onClick={() => setCount(count + 1)}> +1 누르기 </button> </div> ); }

현재 카운트: 0

⚠️ 직접 수정 금지 count = count + 1 처럼 변수를 직접 바꾸면 리액트가 화면을 바꾸지 않습니다. 반드시 setCount(새로운값) 함수를 통해서만 값을 변경해야 합니다.

5. 실전 종합: 좋아요 버튼 만들기

State와 조건부 렌더링(삼항 연산자)을 섞으면 인스타그램 같은 '좋아요' 토글 버튼을 쉽게 만들 수 있습니다.

function LikeButton() { // 처음엔 좋아요를 안 누른 상태(false) const [isLiked, setIsLiked] = useState(false); // 클릭 시 현재 상태를 반대로 뒤집음 (!isLiked) const handleLike = () => { setIsLiked(!isLiked); }; return ( <button onClick={handleLike} {/* isLiked가 true면 빨강, 아니면 회색 */} style={{ color: isLiked ? "red" : "gray" }} > {/* true면 꽉찬 하트, false면 빈 하트 */} {isLiked ? "♥ 좋아요 취소" : "♡ 좋아요"} </button> ); }
