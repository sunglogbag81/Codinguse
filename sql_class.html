Chapter 8. 데이터베이스의 심장

SQL 데이터 다루기 (Full Course)

서버가 기억력을 갖기 위해 사용하는 엑셀의 초진화 버전입니다. SQL은 데이터에게 "원하는 것을 정확하게 찾아와!"라고 명령을 내리는 전 세계 공통어입니다.

1. 데이터베이스와 테이블

데이터베이스는 커다란 파일 서랍장이고, 그 안에 들어있는 엑셀 시트 하나하나를 테이블(Table)이라고 부릅니다. 세로줄은 컬럼(Column, 속성), 가로줄은 로우(Row, 데이터 한 줄)입니다.

-- users 테이블을 처음으로 생성하는 명령 CREATE TABLE users ( id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(50), age INT, grade VARCHAR(10) );
▶ Table Schema Created: `users`
ColumnTypeKey
idINTPRI
nameVARCHAR
ageINT
gradeVARCHAR
💡 Primary Key (기본키)란? 주민등록번호나 학번처럼, 수백만 개의 데이터 중에서 이 데이터를 유일하게 식별할 수 있는 고유한 값입니다. 데이터베이스에서 가장 중요한 개념입니다.

2. 데이터 조회하기 (SELECT, FROM, WHERE)

SQL에서 가장 많이 쓰는, 데이터를 검색하는 명령입니다. SELECT(무엇을), FROM(어디서), WHERE(어떤 조건으로) 세 가지만 알면 90%의 질문을 던질 수 있습니다.

모든 데이터 가져오기 vs 조건 걸기

별표(*)는 '모든 컬럼'을 의미합니다. WHERE 뒤에는 파이썬이나 자바처럼 비교 연산자를 써서 조건을 줍니다.

-- 1. users 테이블의 모든 컬럼(*)과 모든 데이터를 가져와라! SELECT * FROM users; -- 2. 나이가 20살 이상인 사람의 이름과 등급만 가져와라! SELECT name, grade FROM users WHERE age >= 20;
▶ Query Result (2 rows returned)
namegrade
홍길동VIP
김철수GOLD

3. 정렬과 개수 제한 (ORDER BY, LIMIT)

게시판의 최신 글을 불러오거나, 게임 랭킹 1~3위를 보여줄 때 필수적으로 사용하는 명령어입니다.

-- 나이(age)를 기준으로 내림차순(DESC) 정렬하고, -- 위에서부터 딱 3명(LIMIT 3)만 가져와라! SELECT * FROM users ORDER BY age DESC LIMIT 3;
▶ Top 3 Oldest Users
idnameage
3박영희35
1홍길동25

💡 DESC 와 ASC DESC(Descending)는 큰 숫자부터 작아지는 내림차순(최신순, 높은점수순)이고, ASC(Ascending)는 작은 숫자부터 커지는 오름차순입니다. (ASC는 기본값이라 생략 가능합니다.)

4. 데이터 조작하기 (INSERT, UPDATE, DELETE)

검색을 벗어나, 테이블에 새로운 데이터를 밀어 넣거나 기존 데이터를 수정하고 지우는 CRUD 작업입니다.

WHERE 조건을 잊으면 대참사가 일어납니다!

UPDATE와 DELETE를 할 때는 반드시 WHERE 절을 적어 특정 데이터만 지정해야 합니다. 안 그러면 모든 유저의 데이터가 날아가거나 덮어씌워집니다.

-- 1. 데이터 추가 (Create) INSERT INTO users (name, age, grade) VALUES ('이데이터', 19, 'NEW'); -- 2. 데이터 수정 (Update) -- 아이디가 1인 유저의 등급을 VVIP로 바꿔라! UPDATE users SET grade = 'VVIP' WHERE id = 1; -- 3. 데이터 삭제 (Delete) -- 나이가 10살 미만인 유저를 지워라! DELETE FROM users WHERE age < 10;
▶ Query Executed Successfully
1 row inserted.
1 row updated.
0 rows deleted.

5. 테이블 연결하기 (JOIN) ★가장 중요

데이터베이스를 강력하게 만드는 이유입니다. 유저 정보 테이블과 주문 내역 테이블을 분리해 두었다가, 필요할 때 공통된 연결고리(id)를 통해 하나로 합쳐서 보여주는 기술입니다.

-- users 테이블의 이름과 orders 테이블의 상품명을 같이 보고 싶다! SELECT users.name, orders.product FROM users -- orders 테이블을 옆에 가져다 붙여라 (JOIN) JOIN orders -- 단, users의 id와 orders의 user_id가 같은 것끼리만 (ON) ON users.id = orders.user_id;
▶ Joined Query Result
name (users)product (orders)
홍길동맥북 프로
홍길동매직 마우스
김철수아이패드
💡 Foreign Key (외래키)란? 위 예시에서 orders 테이블의 user_id처럼, 다른 테이블의 기본키(Primary Key)를 가리키는 속성을 외래키라고 합니다. 이 외래키 덕분에 테이블 간에 거미줄처럼 관계를 맺을 수 있습니다.
