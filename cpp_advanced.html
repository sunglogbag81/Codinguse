C++ 심화 (Advanced)

C언어의 포인터 지옥에서 벗어나 모던 C++의 세계로 진입합니다. 메모리 누수를 원천 차단하는 스마트 포인터부터 멀티스레딩까지, 하드웨어 성능을 극한으로 끌어올리는 기술을 마스터합니다.

1. 스마트 포인터 (Smart Pointers)

C언어에서 malloc/free를, 구버전 C++에서 new/delete를 쓰며 발생하던 메모리 누수(Memory Leak)를 완벽하게 해결한 모던 C++의 핵심 기술입니다. 객체가 사용되지 않으면 자동으로 메모리를 해제합니다.

std::unique_ptr (독점 소유권)

오직 단 하나의 포인터만이 객체를 가리킬 수 있게 강제하여 가장 안전하고 빠릅니다.

#include <iostream> #include <memory> // 스마트 포인터를 위한 헤더 class Player { public: Player() { std::cout << "Player 생성됨!\n"; } ~Player() { std::cout << "Player 소멸됨! (메모리 자동 해제)\n"; } void Attack() { std::cout << "공격!\n"; } }; int main() { { // new 대신 std::make_unique를 사용합니다. std::unique_ptr<Player> p1 = std::make_unique<Player>(); p1->Attack(); // std::unique_ptr<Player> p2 = p1; // ❌ 컴파일 에러! (복사 불가) } // 블록이 끝나는 순간 p1이 자동으로 소멸(delete)됩니다! std::cout << "메인 함수 종료\n"; return 0; }
> Terminal - ./main
Player 생성됨!
공격!
Player 소멸됨! (메모리 자동 해제)
메인 함수 종료
💡 std::shared_ptr 이란? 만약 여러 포인터가 하나의 객체를 공유해야 한다면 std::shared_ptr를 사용합니다. 내부적으로 참조 카운트(Reference Count)를 세어, 나를 가리키는 포인터가 0개가 되는 순간 객체를 파괴합니다.

2. 이동 의미론과 우측값 참조 (Move Semantics)

엄청나게 큰 데이터를 다른 변수에 넘겨줄 때, 데이터 전체를 '복사(Copy)'하면 성능이 뚝 떨어집니다. C++11부터는 원본 데이터의 소유권만 훔쳐오는 '이동(Move)'이 가능해졌습니다.

std::move

#include <iostream> #include <string> #include <vector> int main() { std::string str1 = "엄청나게 긴 텍스트 데이터라고 가정합시다."; // 1. 복사 (Copy) - 메모리가 두 배로 들고 느림 std::string str2 = str1; // 2. 이동 (Move) - str1의 메모리 주소만 str3로 쓱싹 가져옴! // 이제 str1은 텅 빈 껍데기가 됩니다. std::string str3 = std::move(str1); std::cout << "str1: [" << str1 << "]\n"; // 비어있음 std::cout << "str3: [" << str3 << "]\n"; // 데이터가 완벽히 넘어옴 return 0; }

3. 템플릿 (Templates) 메타프로그래밍

자바(Java)의 제네릭(Generic)과 비슷하지만 훨씬 강력합니다. "타입(자료형)을 변수처럼 취급하여, 코드를 찍어내는 코드를 작성"하는 기법으로, STL(표준 템플릿 라이브러리)의 근간입니다.

#include <iostream> // 템플릿 선언: T 라는 임의의 타입을 사용하겠다! template <typename T> T GetMax(T a, T b) { return (a > b) ? a : b; } int main() { // int를 넣으면 int 전용 GetMax가 컴파일 타임에 생성됨 int i = GetMax<int>(10, 20); // double을 넣으면 double 전용 GetMax가 생성됨 double d = GetMax<double>(3.14, 5.55); std::cout << "정수 최대값: " << i << "\n"; std::cout << "실수 최대값: " << d << "\n"; return 0; }
> Terminal - ./main
정수 최대값: 20
실수 최대값: 5.55

4. 람다 표현식 (Lambda Expressions)

C++11에서 도입된 이름 없는 함수(익명 함수)입니다. std::sort 나 std::find_if 같은 알고리즘 함수에 비교 조건을 넘겨줄 때 코드를 극적으로 간결하게 만들어줍니다.

[캡처](매개변수) -> 반환타입 { 본문 }

#include <iostream> #include <vector> #include <algorithm> int main() { std::vector<int> numbers = {5, 2, 8, 1, 9}; int limit = 4; // 1. 람다를 이용한 커스텀 정렬 (내림차순) // [](캡처 없음), (int a, int b) 매개변수 std::sort(numbers.begin(), numbers.end(), [](int a, int b) { return a > b; }); // 2. 외부 변수(limit) 캡처하기 // [limit] 처럼 대괄호 안에 쓰면 외부 변수를 람다 안에서 쓸 수 있습니다. auto it = std::find_if(numbers.begin(), numbers.end(), [limit](int n) { return n < limit; }); if (it != numbers.end()) { std::cout << limit << "보다 작은 첫 번째 숫자: " << *it << "\n"; } return 0; }
💡 캡처(Capture) 블록의 비밀 [&]를 쓰면 외부의 모든 변수를 참조(Reference)로 가져와 내부에서 값을 수정할 수 있고, [=]를 쓰면 복사(Value)해서 가져옵니다.

5. 모던 멀티스레딩 (Concurrency)

C언어 시절의 복잡한 운영체제 API(pthread)를 몰라도, C++11부터는 표준 라이브러리(<thread>)를 통해 쉽게 멀티 코어 프로그래밍을 할 수 있습니다.

std::thread 와 std::mutex

여러 스레드가 동시에 변수에 접근하여 데이터가 꼬이는 Race Condition(경쟁 상태)을 막기 위해 mutex(자물쇠)를 사용합니다.

#include <iostream> #include <thread> #include <mutex> int shared_counter = 0; std::mutex mtx; // 자물쇠 객체 void IncreaseCounter() { for(int i = 0; i < 10000; i++) { // lock_guard: 블록을 벗어나면 자동으로 자물쇠가 풀림 (안전함) std::lock_guard<std::mutex> lock(mtx); shared_counter++; } } int main() { // 두 개의 스레드(일꾼)를 동시에 출격시킴 std::thread t1(IncreaseCounter); std::thread t2(IncreaseCounter); // join(): 스레드들이 일을 다 마칠 때까지 메인 함수가 기다려줌 t1.join(); t2.join(); std::cout << "최종 카운트: " << shared_counter << "\n"; return 0; }
> Terminal - ./main
