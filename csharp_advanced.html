C# 심화 (Advanced)

C#의 진짜 힘은 압도적인 생산성과 유연함에 있습니다. 데이터 질의의 마법인 LINQ부터, 옵저버 패턴의 정수인 이벤트, 그리고 게임 성능을 좌우하는 메모리 풀링까지 완벽하게 정복합니다.

1. 데이터 질의의 마법: LINQ (Language Integrated Query)

수십 개의 리스트와 배열 속에서 특정 조건을 만족하는 데이터를 찾을 때, 복잡한 for문과 if문을 쓸 필요가 없습니다. SQL 쿼리를 작성하듯 C# 코드 한 줄로 데이터를 가공하는 강력한 기능입니다.

using System; using System.Linq; // LINQ 사용을 위한 네임스페이스 using System.Collections.Generic; class Player { public string Name; public int Level; } class Program { static void Main() { List<Player> players = new List<Player> { new Player { Name = "전사", Level = 15 }, new Player { Name = "마법사", Level = 40 }, new Player { Name = "궁수", Level = 25 } }; // LINQ: 레벨이 20 이상인 플레이어를 찾아 레벨 내림차순으로 정렬 후 이름만 가져옴 var highLevelNames = players .Where(p => p.Level >= 20) .OrderByDescending(p => p.Level) .Select(p => p.Name) .ToList(); foreach (var name in highLevelNames) { Console.WriteLine(name); // "마법사", "궁수" 순으로 출력 } } }
💡 지연 실행 (Lazy Evaluation) LINQ의 Where나 Select는 그 즉시 코드를 실행하지 않고 계획표만 짜둡니다. 최종적으로 ToList()나 foreach를 만났을 때 비로소 한 번에 연산을 수행하여 성능을 극대화합니다.

2. 델리게이트와 이벤트 (옵저버 패턴)

"보스가 죽으면 UI를 업데이트하고, 업적을 달성하고, 문을 열어라." 이 코드를 보스 스크립트 안에 다 적으면 코드가 스파게티가 됩니다. 이벤트(Event)를 사용하면 보스는 "나 죽었어!"라고 방송(Invoke)만 하고, 다른 스크립트들이 알아서 반응하게 만들 수 있습니다.

Action을 활용한 Event 구현

using System; class Boss { // 매개변수와 반환값이 없는 함수들을 담을 수 있는 변수(이벤트) public event Action OnBossDied; public void Die() { Console.WriteLine("보스가 쓰러졌습니다!"); // 이벤트에 등록된 모든 함수들을 한 번에 실행 (?. 는 Null 체크) OnBossDied?.Invoke(); } } class Program { static void Main() { Boss dragon = new Boss(); // 이벤트에 구독(구독 취소는 -=) dragon.OnBossDied += () => Console.WriteLine("[UI] 승리 화면 표시"); dragon.OnBossDied += () => Console.WriteLine("[업적] 드래곤 슬레이어 달성"); // 보스 사망 시 등록된 모든 함수가 연쇄적으로 실행됨 dragon.Die(); } }
> Terminal - dotnet run
보스가 쓰러졌습니다!
[UI] 승리 화면 표시
[업적] 드래곤 슬레이어 달성

3. 비동기 프로그래밍 (async / await)

서버에서 아이템 정보를 불러오거나 무거운 파일을 읽을 때, 일반 코드를 쓰면 그동안 게임 화면(UI)이 완전히 멈춰버립니다(프리징). 비동기 처리를 사용하면 백그라운드에서 일을 처리하고 끝날 때까지 우아하게 기다릴 수 있습니다.

using System; using System.Threading.Tasks; // Task를 위한 네임스페이스 class Program { // 비동기 함수는 반환 타입을 Task로 설정하고 async 키워드를 붙입니다. static async Task LoadGameDataAsync() { Console.WriteLine("서버에서 데이터를 불러오는 중..."); // 3초 동안 대기 (이 동안 프로그램이 멈추지 않고 다른 일을 할 수 있음!) await Task.Delay(3000); Console.WriteLine("데이터 로드 완료!"); } static async Task Main() { // 비동기 함수 호출 시 await를 붙여 결과가 올 때까지 기다림 await LoadGameDataAsync(); } }

4. 확장 메서드 (Extension Methods)

우리가 수정할 수 없는 기존 라이브러리의 클래스(예: string이나 유니티의 Transform)에 마치 원래 있었던 메서드처럼 새로운 기능을 끼워 넣는 마법입니다.

using System; // 확장 메서드는 반드시 'static class' 안에 선언해야 합니다. public static class StringExtensions { // 첫 번째 매개변수 앞에 'this'를 붙이면, 해당 타입의 확장 메서드가 됩니다. public static bool IsAwesome(this string str) { return str.Contains("최고"); } } class Program { static void Main() { string myText = "C#은 정말 최고야!"; // 마치 원래 string에 포함된 기능처럼 사용할 수 있음! bool result = myText.IsAwesome(); Console.WriteLine(result); // True 출력 } }
> Terminal - dotnet run
True

5. (유니티 심화) 오브젝트 풀링 (Object Pooling)

게임에서 총알 100발을 쏠 때마다 메모리에 새롭게 만들고(Instantiate), 맞으면 삭제(Destroy)하면 치명적인 가비지 컬렉션(GC) 스파이크(프레임 드랍)가 발생합니다.
대신, 총알을 미리 100개 만들어 웅덩이(Pool)에 담아두고 꺼내 썼다가, 다 쓰면 끄고(SetActive(false)) 다시 반납하는 기법이 오브젝트 풀링입니다.

using UnityEngine; using System.Collections.Generic; public class BulletPool : MonoBehaviour { public GameObject bulletPrefab; private Queue<GameObject> pool = new Queue<GameObject>(); void Start() { // 게임 시작 시 미리 총알 50개를 만들어두고 숨겨둠 for (int i = 0; i < 50; i++) { GameObject obj = Instantiate(bulletPrefab); obj.SetActive(false); pool.Enqueue(obj); // 풀(Queue)에 넣기 } } // 총알 쏘기 (새로 만들지 않고 풀에서 하나 꺼냄) public GameObject GetBullet() { if (pool.Count > 0) { GameObject obj = pool.Dequeue(); obj.SetActive(true); return obj; } return null; // 풀이 비었으면 못 쏨 } // 총알이 어딘가 부딪혀서 없어질 때 (삭제하지 않고 다시 풀에 반납) public void ReturnBullet(GameObject obj) { obj.SetActive(false); pool.Enqueue(obj); } }
💡 메모리 최적화의 기본 유니티를 비롯한 거의 모든 게임 엔진에서 Instantiate와 Destroy는 가장 무거운 연산입니다. 탄막 슈팅, 파티클 이펙트, 적 몬스터 스폰 등 자주 생성되고 사라지는 모든 오브젝트에는 무조건 풀링 패턴을 적용해야 합니다.
