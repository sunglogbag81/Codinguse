Initial Prompt

"파이썬으로 할 일 관리 프로그램을 만들어줘. 추가, 목록 보기, 완료 처리 기능이 필요해."

Build & Fix Loop
Part 1

파이썬 기초는 이 네 가지를 먼저 잡으면 된다

Step 1

변수 / 자료형

데이터를 담는 방식과 글자, 숫자의 차이를 먼저 이해합니다.

Step 2

연산자

계산과 비교를 통해 프로그램이 판단할 재료를 만듭니다.

Step 3

조건문 / 반복문

상황에 따라 나누고, 필요한 만큼 반복하는 구조를 배웁니다.

Step 4

직접 실습

CodeUp과 백준으로 눈으로 본 내용을 손으로 익혀야 실력이 남습니다.

파이썬 기초 1

변수와 자료형: 값을 담는 상자부터 이해한다

핵심 요약

이것만 기억하면 시작할 수 있다

•변수: 데이터를 담아 두는 이름표다
•문자열(str): 글자 데이터, 보통 ""를 쓴다
•정수(int): 계산 가능한 숫자 데이터다
코드 스니펫

할당과 출력

name = "김코딩" age = 16 print(name, age)
기억: =는 수학의 같다가 아니라 오른쪽 값을 왼쪽 변수에 넣는 뜻입니다.
파이썬 기초 2

연산자와 조건문: 계산하고, 비교하고, 판단한다

연산자

계산과 비교의 재료

a = 10 b = 3 print(a + b) print(a % b) print(a > b) print(a == b)
조건문

위에서 아래로 경우를 나눈다

score = 85 if score >= 90: print("A등급") elif score >= 80: print("B등급") else: print("C등급")
비교 결과 True / False가 조건문의 재료가 됩니다
파이썬 기초 3

반복문은 같은 작업을 복붙하지 않게 해 준다

기본 구조

for + range

for i in range(3): print(i, "번째 반복입니다")

컴퓨터는 보통 0부터 세기 때문에 range(3)은 0, 1, 2 세 번 반복합니다.

범위 지정

1부터 5까지 출력

for i in range(1, 6): print(i)
•시작 값은 포함합니다
•끝 값은 포함하지 않습니다
•그래서 결과는 1, 2, 3, 4, 5 입니다
파이썬 기초 4

실전에서는 조건문과 반복문을 같이 읽는 경우가 많다

예시 코드

1부터 10까지 중 짝수만 찾기

for i in range(1, 11): if i % 2 == 0: print(i, "은(는) 짝수!")
읽는 순서

바깥에서 안쪽으로 읽는다

1먼저 for문으로 몇 번 도는지 본다
2그 다음 if문으로 어떤 경우만 통과하는지 본다
3조건이 참일 때만 안쪽 코드가 실행된다
파이썬 기초 5

자주 헷갈리는 포인트만 잡아도 초반 실수가 크게 줄어든다

if

경우를 나눠야 할 때는 조건문부터 떠올리면 됩니다.

for

같은 걸 여러 번 해야 한다면 반복문일 가능성이 큽니다.

들여쓰기

파이썬은 띄어쓰기로 실행 범위를 구분하므로 매우 중요합니다.

= vs ==

=은 대입, ==은 비교입니다.

눈으로 읽고 끝내면 내 코드가 아닙니다. 직접 타이핑해야 실력이 남습니다
실습 연결

기초 문법은 문제 풀이와 함께 갈 때 훨씬 빨리 익는다

CodeUp

문법 체력 키우기

✓조건문 기초: 1151 ~ 1165번
✓반복문 기초: 1251 ~ 1261번
✓배운 문법을 바로 손으로 확인하기 좋다
백준

조금 익숙해지면 사고력 훈련

•입력과 출력 형식을 스스로 해석해야 한다
•처음에는 브론즈, 실버 기초 문제부터 가볍게 시작한다
•문법을 넘어서 문제 해결력 자체를 키워 준다
Part 2

이제 문법을 바탕으로, AI를 써서 실제 문제를 해결해 본다

왜 연결하나

문법만 아는 상태를 넘기기 위해

기초 문법을 안다는 것과 실제로 뭔가를 만드는 것은 다릅니다. AI는 그 간격을 줄여 줍니다.

핵심 방식

작게 묻고, 실행하고, 다시 고친다

한 번에 완벽을 기대하지 않고, 짧은 반복으로 결과물을 개선해 가는 흐름이 중요합니다.

이제부터

실전 파트 시작

문제 설정, 프롬프트, 실행, 오류 수정, 도구 선택까지 이어서 보겠습니다.

오늘 할 것

설명보다, 직접 해결하는 흐름만 봅니다

1. No Talk, Just Action

설명 안 한다

이론적인 장점 이야기 길게 안 합니다. 바로 에디터를 켜고 실전으로 갑니다.

2. Step-by-Step

해결 흐름을 본다

문제 설정 → 요청 → 실행 → 오류 직면 → 수정 과정을 낱낱이 봅니다.

3. Prompt Tuning

프롬프트 차이를 익힌다

결과가 도대체 왜 달라지는지, Before & After 비교로 감각을 잡습니다.

문제 설정

바이브 코딩은 무엇을 만들지 정하는 순간부터 이미 시작된다

처음부터 큰 주제를 잡으면 프롬프트도 흐려지고 결과도 산만해집니다. 문제 설정 단계에서는 "지금 당장 만들 수 있는 가장 작은 문제"로 줄이는 것이 가장 중요합니다.

좋은 문제 설정

처음 문제는 보통 이런 조건이 좋다

1설명 한두 문장으로 말할 수 있을 만큼 작다
2핵심 기능이 2~3개 정도로 제한되어 있다
3실행했을 때 결과를 바로 확인할 수 있다
4막히면 다음 수정 요청으로 이어 가기 쉽다
나쁜 시작 vs 좋은 시작

처음부터 이렇게 차이가 난다

나쁜 시작: 게임 하나 만들어줘 / 사이트 하나 만들어줘
좋은 시작: 파이썬으로 할 일 추가, 목록 보기, 완료 처리만 되는 작은 프로그램 만들어줘
핵심: 처음 문제를 작게 잡아야 이후 수정도 쉬워진다
실수 예시

이렇게 하면 망한다

나쁜 프롬프트

요구사항이 비어 있음

파이썬으로 프로그램 만들어줘
✕무슨 프로그램인지 전혀 모름
✕어떤 기능이 필요한지 없음
✕난이도 제한도 없음
나오는 결과

애매하고 쓸모없는 답이 돌아온다

"간단한 예제로 계산기 프로그램을 만들었습니다. 더 원하시는 기능이나 프로그램 종류가 있다면 말씀해 주세요."
내가 원한 할 일 관리 프로그램이 아님
처음부터 다시 설명해야 해서 시간 낭비 발생
1차 요청

좋은 시작은 요구사항을 '작고 정확하게' 말하는 것이다

올바른 프롬프트

이렇게 요청한다

파이썬으로 할 일 관리 프로그램을 만들어줘. 기능은 1) 추가 2) 목록 보기 3) 완료 처리야. 초보자 수준으로, 함수로 나눠주고, 주석도 넣어줘.
체크포인트

퀄리티가 높아진 이유

✓목적: 무엇을 만들지 분명함 (할 일 관리)
✓범위: 기능이 정확히 3개로 제한됨
✓난이도: 초보자 수준이라고 선을 그음
✓출력: 함수 분리와 주석이라는 형식을 지정함
실행 결과

코드는 받았는데, 실행해 보니 불편한 점이 보인다

실행 테스트

발견된 문제점

완료된 항목과 미완료 항목이 UI에서 구분되지 않음
터미널 출력 형식이 복잡해서 읽기 불편함
기능 작동은 되지만, 실사용하기엔 부족함
핵심 마인드셋

멈추지 않고 다시 요청한다

"AI 코딩 별로네" 하고 닫는 것이 아니라, "어디가 불편한지"를 정확히 캐치해서 다음 프롬프트의 재료로 쓰는 것이 핵심!
핵심 단계

피드백 루프: 이렇게 고친다

수정 프롬프트

문제를 그대로 전달하기

이 코드 실행했더니 이런 문제가 있어. 1. 완료된 항목을 [V] 표시 등으로 구분할 수 있게 해줘. 2. 콘솔 출력 코드를 더 간단하고 보기 좋게 정리해줘.
기억할 것

한 번에 완벽 X, 계속 개선 O

✓실행 결과를 눈으로 보고 문제를 찾는다
✓불편한 점을 사람에게 말하듯 구체적으로 말한다
✓조금씩 고치면서 내가 원하는 완벽한 형태로 간다
프로세스

실전 바이브 코딩 흐름은 이 한 줄로 정리된다

파트 2

나쁜 프롬프트 vs 좋은 프롬프트

나쁜 예

너무 막연함

게임 만들어줘
✕장르, 규칙, 그래픽 유무 전혀 없음
✕내 코딩 수준에 맞지 않는 코드 폭격
✕실행 방법조차 알려주지 않음
좋은 예

실행 가능한 구조적 요청

파이썬 터미널에서 실행되는 숫자 맞히기 게임을 만들어줘. 초보자 수준으로, 함수는 2~3개 정도만 쓰고, 주석과 코드 실행 방법을 반드시 포함해줘.
✓장르(숫자 맞히기)와 환경(터미널) 명시
✓난이도와 코드 구조(함수 개수) 제한
✓주석과 가이드(실행 방법) 요구
체크리스트

원샷(One-shot) 프롬프트 필수 5요소

1. 목표 (What)

게임인지, 관리 도구인지, 크롤러인지 명확하게.

2. 기능 (Features)

초기 버전은 가장 핵심적인 기능 2~3개로 좁히기.

3. 난이도 (Level)

자신의 수준(초급/중급)에 맞춰 코드 복잡도 조절.

4. 출력 (Output)

코드만? 주석 포함? 설명 포함? 형식을 강제하기.

5. 스타일 (Style)

함수 분리, 클래스 사용, 단일 파일 등 구조적 조건 붙이기.

💡 마법의 공식

목표 + 기능 + 난이도 + 출력 형태 + 스타일

복붙용 템플릿

상황별 바로 써먹는 프롬프트 치트시트

1. 기획/아이디어

무엇을 만들지 막막할 때

파이썬 기초 문법만 아는 상태야. 주말 동안 2시간 만에 만들 수 있는 흥미로운 미니 프로젝트 아이디어 5개를 추천해줘. 각 기능 요건과 난이도를 표로 정리해줘.
2. 초기 코드 생성

첫 뼈대를 잡을 때

파이썬으로 CLI 기반 할 일 관리 앱을 짜줘. CRUD(추가/읽기/수정/삭제) 중 '추가, 읽기'만 우선 구현해. 초보자가 이해하도록 주석을 아주 상세히 달아줘.
3. 디버깅/오류 해결

에러가 나고 막혔을 때

[에러 메시지 복사/붙여넣기] 이 코드를 실행했더니 위와 같은 에러가 났어. 발생 원인을 1줄로 쉽게 설명하고, 수정한 전체 코드를 다시 줘.
4. 확장 및 리팩토링

기능을 추가하거나 코드를 개선할 때

현재 코드 완벽하게 잘 돌아가. 여기에 '완료된 항목 숨기기' 기능만 추가해줘. 기존 아키텍처는 건드리지 말고 바뀐 함수만 설명해.
파트 3

바이브 코딩도 도구를 잘 고르면 훨씬 강해진다

처음에는 가장 쉬운 웹 AI로 시작하고, 익숙해지면 실행형 도구와 에디터형 도구로 확장하는 방식이 가장 안정적입니다.

Web AI

ChatGPT / Gemini

질문하고, 설명 듣고, 코드 받기 가장 쉽습니다. 처음 배우는 사람에게 진입장벽이 낮습니다.

Run Fast

Replit

설치 없이 코드 작성과 실행을 한곳에서 처리하기 좋아 수업, 실습, 공유용으로 편합니다.

Editor Assist

GitHub Copilot

VS Code에서 자동완성과 제안을 받아 직접 타이핑하며 배우기 좋습니다.

Level Up

Codex / Claude Code

기존 프로젝트를 읽고 수정하거나, 더 긴 작업 흐름을 맡길 때 강점이 커집니다.

추천 흐름: 웹 AI → Replit/Copilot → 더 강한 작업형 도구
입문 추천

무료 또는 접근하기 쉬운 도구부터 시작하면 된다

가장 무난

ChatGPT / Gemini 웹 버전

✓설명 듣기, 코드 생성, 오류 질문이 매우 쉽다
✓브라우저만 있으면 바로 시작 가능하다
✓처음 배우는 사람에게 가장 부담이 적다
실행까지

Replit / GitHub Copilot

✓Replit은 설치 없이 실습하고 공유하기 좋다
✓Copilot은 에디터 안에서 직접 코드를 치며 배우기 좋다
✓학생용 무료 혜택은 발표 때 따로 안내하기 좋다
입문자 추천 조합
설명 듣기: ChatGPT 또는 Gemini
바로 실행: Replit
에디터 보조: Copilot
주의

처음부터 비싼 도구가 정답은 아니다
