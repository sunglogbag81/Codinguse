1. JVM 아키텍처와 메모리 관리

자바 개발자라면 코드가 실행되는 환경인 JVM(Java Virtual Machine)의 내부를 알아야 합니다. 특히 메모리 영역과 **가비지 컬렉션(GC)**의 원리를 이해하는 것이 성능 최적화의 첫걸음입니다.

Runtime Data Areas

  • Stack Area: 스레드마다 생성되며, 지역 변수와 메서드 호출 정보가 저장됩니다.
  • Heap Area: 모든 객체(인스턴스)가 저장되는 곳입니다. GC의 주요 대상입니다.
  • Method Area: 클래스 정보, 상수가 저장되는 공유 영역입니다.
💡 Garbage Collection (GC) Heap 영역에서 더 이상 참조되지 않는 객체들을 추적하여 자동으로 메모리에서 해제하는 프로세스입니다. 개발자가 직접 메모리를 해제할 필요가 없게 해주지만, 실행 시 앱이 잠시 멈추는 'Stop-the-world' 현상을 관리하는 것이 핵심입니다.

2. 제네릭 (Generics)

다양한 타입의 객체들을 다루는 메서드나 클래스에서 컴파일 시 타입 체크를 해주는 기능입니다. 런타임 에러(ClassCastException)를 예방하는 가장 강력한 도구입니다.

// T는 타입 파라미터 (나중에 결정됨) public class Box<T> { private T item; public void setItem(T item) { this.item = item; } public T getItem() { return item; } } // 사용 시 타입을 지정 Box<String> strBox = new Box<>(); strBox.setItem("Hello"); // strBox.setItem(10); // ❌ 컴파일 에러 발생!
> 컴파일러가 지켜보고 있습니다.

3. 람다와 함수형 인터페이스

자바 8부터 도입된 **함수형 프로그래밍**의 정수입니다. 익명 함수를 통해 코드를 간결하게 만들고 가독성을 획기적으로 높입니다.

Lambda Expression

메서드가 하나만 있는 인터페이스(함수형 인터페이스)를 구현할 때 사용합니다.

// 과거의 익명 클래스 방식 new Thread(new Runnable() { @Override public void run() { System.out.println("Thread start!"); } }).start(); // 람다(Lambda) 방식 new Thread(() -> System.out.println("Thread start!")).start();

4. 스트림(Stream) API ★중요

컬렉션(List, Map 등)의 데이터를 **함수형 스타일**로 처리하는 강력한 API입니다. 필터링, 매핑, 정렬 등을 파이프라인 형태로 연결해 직관적인 코드를 짤 수 있습니다.

List<String> names = Arrays.asList("Kim", "Lee", "Park", "Choi"); // 'P'로 시작하는 이름만 찾아 대문자로 바꾸어 출력 names.stream() .filter(name -> name.startsWith("P")) .map(String::toUpperCase) .forEach(System.out::println);

5. 멀티스레드와 동시성 제어

하나의 프로그램에서 여러 작업을 동시에 수행하는 기술입니다. 자바는 언어 차원에서 멀티스레드를 강력하게 지원하지만, **공유 자원 문제(Race Condition)**를 해결하는 것이 가장 까다롭고 중요합니다.

synchronized 와 동기화

여러 스레드가 동시에 같은 변수를 수정하려고 할 때, 한 번에 하나의 스레드만 접근하도록 잠금(Lock)을 거는 과정입니다.

public class Counter { private int count = 0; // synchronized 키워드로 임계 영역(Critical Section) 설정 public synchronized void increment() { count++; } public int getCount() { return count; } }

6. Optional (Null Safety)

자바 개발자의 영원한 숙적 NullPointerException을 방어하기 위해 도입된 Wrapper 클래스입니다. "값이 있을 수도 있고 없을 수도 있다"를 명시적으로 표현합니다.

Optional<String> opt = Optional.ofNullable(getNullableName()); // 값이 있으면 출력하고, 없으면 "Default" 출력 System.out.println(opt.orElse("Default Name")); // 값이 있을 때만 특정 동작 수행 opt.ifPresent(name -> System.out.println("Name is " + name));
