C 프로그래밍 심화 (Advanced)

포인터의 공포를 이겨냈다면, 이제 메모리의 완벽한 통제권을 쥘 차례입니다. 동적 할당, 이중 포인터, 자료구조의 근간을 직접 설계하며 C언어의 진정한 힘을 깨워봅시다.

1. 동적 메모리 할당 (Heap 영역의 지배자)

일반적인 변수나 배열은 크기가 고정되어 있고 함수가 끝나면 사라집니다(Stack 영역). 실행 중에 사용자가 입력한 크기만큼 자유롭게 메모리를 할당받고 해제하려면 Heap 영역과 malloc을 사용해야 합니다.

malloc, calloc, realloc 그리고 free

메모리를 빌렸으면(할당), 다 쓴 뒤에는 반드시 운영체제에 반납(free)해야 합니다. 안 그러면 메모리 누수(Memory Leak)로 컴퓨터가 뻗어버립니다.

#include <stdio.h> #include <stdlib.h> // 메모리 동적 할당을 위한 헤더 int main() { int size; printf("몇 명의 점수를 입력할까요? "); scanf("%d", &size); // 1. 입력받은 size만큼 메모리(Heap) 할당 // malloc은 void* 를 반환하므로 int* 로 형변환 해줍니다. int *scores = (int*)malloc(size * sizeof(int)); if (scores == NULL) { printf("메모리 할당 실패!\n"); return 1; } // 2. 일반 배열처럼 자유롭게 사용 for(int i = 0; i < size; i++) { scores[i] = (i + 1) * 10; } // 3. ★가장 중요★ 다 쓴 메모리 반납 free(scores); // 4. Dangling Pointer(허공을 가리키는 포인터) 방지를 위해 NULL 처리 scores = NULL; return 0; }
💡 calloc vs malloc malloc은 메모리를 할당만 하고 안의 쓰레기값(Garbage)을 비워주지 않습니다. 반면 calloc은 할당과 동시에 모든 메모리 공간을 0으로 초기화해주는 차이가 있습니다.

2. 심연의 포인터: 이중 포인터와 함수 포인터

포인터가 변수의 주소를 담는다면, 이중 포인터(`**`)는 포인터 변수의 주소를 담습니다. 함수 포인터는 변수가 아닌 '함수 자체'의 주소를 담아 콜백(Callback)을 구현할 때 씁니다.

함수 포인터 (Function Pointer)

#include <stdio.h> // 덧셈과 뺄셈을 하는 일반 함수 두 개 int add(int a, int b) { return a + b; } int sub(int a, int b) { return a - b; } // 함수 포인터를 매개변수로 받는 '계산기' 함수 // "int 두 개를 받고 int를 반환하는 함수면 아무거나 다 들어와!" void calculator(int x, int y, int (*func_ptr)(int, int)) { int result = func_ptr(x, y); printf("결과: %d\n", result); } int main() { // 함수 자체를 매개변수(인자)로 넘깁니다. (자바스크립트의 콜백과 동일) calculator(10, 5, add); // 15 출력 calculator(10, 5, sub); // 5 출력 return 0; }
> Terminal - ./a.out
결과: 15
결과: 5

3. 자료구조의 시작: 구조체와 연결 리스트(Linked List)

C언어에는 자바나 파이썬처럼 크기가 맘대로 변하는 List 객체가 없습니다. 우리가 구조체(struct)와 포인터를 이용해 데이터와 다음 데이터의 주소를 기차처럼 연결해 직접 만들어야 합니다.

#include <stdio.h> #include <stdlib.h> // 자기 자신과 똑같은 구조체를 가리키는 포인터를 가짐 (자기 참조 구조체) typedef struct Node { int data; struct Node* next; // 다음 기차칸의 주소를 저장 } Node; int main() { // 1. 노드(기차칸) 2개 동적 할당 Node* head = (Node*)malloc(sizeof(Node)); Node* second = (Node*)malloc(sizeof(Node)); // 2. 데이터 넣기 head->data = 10; second->data = 20; // 3. 기차칸 연결하기 (head의 다음은 second) head->next = second; second->next = NULL; // 마지막 꼬리칸은 NULL을 가리킴 // 4. 리스트 순회하며 출력하기 Node* curr = head; while(curr != NULL) { printf("%d -> ", curr->data); curr = curr->next; // 다음 칸으로 이동 } printf("NULL\n"); free(head); free(second); return 0; }

4. 비트 연산 (Bitwise Operations)

메모리를 극한으로 아끼고, CPU가 가장 좋아하는 연산을 하려면 0과 1 단위로 쪼개서 조작하는 비트 연산을 알아야 합니다. 아두이노 같은 임베디드, 하드웨어 제어에서 필수입니다.

비트 마스크 (Bit Masking)

#include <stdio.h> // 권한을 비트 자리수로 정의 (1, 2, 4, 8 ...) #define READ_PERM 0x01 // 0001 #define WRITE_PERM 0x02 // 0010 #define EXEC_PERM 0x04 // 0100 int main() { unsigned char my_perm = 0; // 처음엔 아무 권한도 없음 (0000) // 1. 권한 추가: 비트 OR (|) 연산 사용 my_perm = my_perm | READ_PERM; // 0001 이 됨 my_perm |= WRITE_PERM; // 0011 이 됨 (읽기, 쓰기 권한 부여) // 2. 권한 확인: 비트 AND (&) 연산 사용 if (my_perm & WRITE_PERM) { printf("쓰기 권한이 있습니다.\n"); } // 3. 권한 제거: 비트 반전(~)과 AND(&) 사용 my_perm &= ~WRITE_PERM; // 0001 로 돌아감 return 0; }
> Terminal - ./a.out
쓰기 권한이 있습니다.
💡 왜 비트로 권한을 관리하나요? 읽기, 쓰기, 실행 권한을 관리하려면 원래 `bool` 변수 3개가 필요합니다. 하지만 비트 연산을 쓰면 단 **1바이트의 변수 하나**만으로 8가지의 권한 상태를 동시에 조작하고 저장할 수 있어 메모리를 극적으로 아낍니다.

5. 파일 입출력 (File I/O)과 바이너리(Binary)

단순한 텍스트 파일(.txt)을 넘어, 이미지나 게임 세이브 데이터처럼 컴퓨터만 알아볼 수 있는 바이너리 파일(.bin, .dat)을 그대로 메모리(구조체)에 붓거나 빼는 기술입니다.

#include <stdio.h> typedef struct { int level; float hp; } SaveData; int main() { SaveData player1 = {50, 1200.5f}; SaveData loaded_player; // 1. 바이너리 쓰기 모드(wb)로 파일 열기 FILE *fp = fopen("save.dat", "wb"); if (fp) { // 메모리 구조 그대로 하드디스크에 통째로 찍어냄 (fwrite) fwrite(&player1, sizeof(SaveData), 1, fp); fclose(fp); } // 2. 바이너리 읽기 모드(rb)로 파일 열기 fp = fopen("save.dat", "rb"); if (fp) { // 하드디스크의 데이터를 그대로 구조체 메모리에 부어넣음 (fread) fread(&loaded_player, sizeof(SaveData), 1, fp); printf("불러온 데이터 - 레벨: %d, 체력: %.1f\n", loaded_player.level, loaded_player.hp); fclose(fp); } return 0; }
