C# 프로그래밍 (Full Course)

C++의 강력함과 Java의 편리함을 모두 갖춘 언어입니다. 특히 전 세계에서 가장 많이 쓰이는 게임 엔진인 Unity(유니티)를 다루기 위해 반드시 거쳐야 하는 필수 관문입니다.

1. C# 프로그램의 첫인상

C#은 자바(Java)와 뼈대가 매우 비슷합니다. using으로 필요한 도구를 가져오고, class 안에 Main 함수를 만들어 실행합니다.

using System; // 기본 입출력 도구함 namespace GameProject { class Program { // 프로그램이 시작되는 곳 static void Main(string[] args) { // 화면에 글자를 출력하고 줄바꿈(Line) Console.WriteLine("Hello, C#!"); } } }
> Terminal - dotnet run
Hello, C#!

2. 변수와 f의 마법 (float)

C#은 엄격한 자료형을 가집니다. 특히 게임 개발에서는 캐릭터의 위치나 속도를 나타낼 때 소수점(실수)을 엄청나게 많이 쓰는데, C#에서는 이를 위해 float를 주로 사용합니다.

using System; class Program { static void Main() { int hp = 100; // 정수 string name = "용사"; // 문자열 bool isAlive = true; // 참/거짓 // 소수점을 쓸 때는 반드시 숫자 뒤에 'f'를 붙여야 합니다! float speed = 5.5f; // 문자열 보간 (달러 기호 $를 쓰면 변수를 쏙쏙 넣기 편합니다) Console.WriteLine($"{name}의 속도는 {speed}입니다."); } }
> Terminal - dotnet run
용사의 속도는 5.5입니다.
💡 왜 float 뒤에 'f'를 붙이나요? C#에서 소수점을 그냥 쓰면 더 크고 무거운 double(8바이트)로 인식합니다. 하지만 게임에서는 초당 60번씩 수만 개의 숫자를 계산해야 하므로, 가벼운 float(4바이트)를 명시하기 위해 f를 꼭 붙여줍니다.

3. 제어문과 foreach

if문과 for문은 다른 언어와 완전히 같습니다. 하지만 C#에는 배열이나 리스트를 훑어볼 때 정말 편리한 foreach라는 꿀단지 문법이 있습니다.

using System; class Program { static void Main() { // 무기 배열 만들기 string[] weapons = { "단검", "장검", "지팡이" }; // for문보다 훨씬 직관적인 foreach문! // "weapons 안에 있는 각각의(foreach) weapon(무기)에 대하여" foreach (string weapon in weapons) { Console.WriteLine($"획득: {weapon}"); } } }
> Terminal - dotnet run
획득: 단검
획득: 장검
획득: 지팡이

4. 클래스와 프로퍼티 (Properties)

C#은 객체지향 언어입니다. 자바에서는 변수를 숨기고(private) getHp(), setHp() 같은 함수를 일일이 만들었지만, C#은 프로퍼티({ get; set; })라는 기능으로 이 과정을 파격적으로 줄였습니다.

using System; class Player { // 한 줄이면 캡슐화 끝! (프로퍼티 기능) public string Name { get; set; } public int Hp { get; private set; } // 피는 외부에서 조작 금지! // 생성자 (태어날 때 초기화) public Player(string name) { Name = name; Hp = 100; } // 데미지를 받는 메서드 public void TakeDamage(int damage) { Hp -= damage; Console.WriteLine($"{Name}이(가) 맞았습니다! 남은 HP: {Hp}"); } } class Program { static void Main() { Player p = new Player("전사"); // p.Hp = 1000; // ❌ private set 이라 조작 불가능! (핵 방지) p.TakeDamage(30); } }
> Terminal - dotnet run
전사이(가) 맞았습니다! 남은 HP: 70

5. (보너스) 유니티(Unity)에서의 C#

C#을 배우는 가장 큰 이유인 Unity 게임 엔진에서는 프로그램의 뼈대가 조금 다릅니다. Main 함수 대신, 유니티가 알아서 실행해 주는 Start()와 Update() 메서드를 사용합니다.

using UnityEngine; // 유니티 엔진 기능 가져오기 // 유니티의 스크립트는 항상 MonoBehaviour를 상속받습니다. public class PlayerController : MonoBehaviour { public float speed = 5.0f; // Start: 게임이 켜지자마자 딱 1번 실행됨 (초기 세팅) void Start() { Debug.Log("게임이 시작되었습니다!"); } // Update: 게임이 진행되는 동안 초당 60번씩(매 프레임) 계속 실행됨 void Update() { // 매 프레임마다 캐릭터를 앞(Forward)으로 이동시킴 transform.Translate(Vector3.forward * speed * Time.deltaTime); } }
> Unity Console
게임이 시작되었습니다!
(캐릭터가 매 프레임마다 앞으로 이동 중...)
💡 Update() 는 왜 필요한가요? 게임은 동영상과 같습니다. 1초에 60장의 사진(프레임)을 빠르게 넘겨서 움직이는 것처럼 보이게 하죠. Update()는 그 사진이 1장 넘어갈 때마다 키보드 입력을 확인하고 캐릭터를 움직이게 해주는 게임의 심장 박동 역할을 합니다.
