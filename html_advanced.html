HTML 심화 (Advanced)

"HTML은 프로그래밍 언어도 아니잖아?" 라고 말하는 사람들에게 진짜 웹의 깊이를 보여주세요. 검색엔진 최적화(SEO), 웹 접근성, 성능 튜닝, 고급 폼 제어의 모든 것을 담았습니다.

1. 시맨틱 웹과 웹 접근성 (A11y)의 극의

단순히 <div>로 떡칠된 사이트는 구글 봇(Bot)도, 시각장애인의 스크린 리더기도 읽어내지 못합니다. 정보의 구조를 완벽하게 짜는 것이 시맨틱 웹의 핵심입니다.

진짜 시맨틱 구조와 WAI-ARIA

의미에 맞는 태그를 쓰는 것을 넘어, 화면에 보이지 않는 요소들에 역할을 부여하는 ARIA(Accessible Rich Internet Applications) 속성을 활용합니다.

<!-- ❌ 하수들의 HTML --> <div class="header"> <div class="nav">메뉴</div> </div> <div class="button" onClick="close()">X</div> <!-- 🟢 고수들의 HTML --> <header> <!-- nav는 문서의 주요 탐색 링크 그룹을 의미 --> <nav aria-label="메인 메뉴"> <ul> <li><a href="/">홈</a></li> </ul> </nav> </header> <main> <!-- article은 그 자체로 독립적인 콘텐츠 (예: 뉴스 기사) --> <article> <h1>HTML 심화 과정</h1> <!-- section은 연관된 내용들의 묶음 --> <section> <h2>접근성이란?</h2> <p>...</p> </section> </article> <!-- 사이드바, 광고 등 본문과 약간 벗어난 내용 --> <aside>관련 강의 추천</aside> </main> <!-- div를 버튼처럼 쓸 때는 반드시 role과 tabindex를 명시해야 접근성이 보장됨 --> <div role="button" tabindex="0" aria-label="창 닫기"> <!-- 시각적 장식인 아이콘은 스크린리더가 읽지 않도록 무시 --> <i class="icon-close" aria-hidden="true"></i> </div>
💡 WAI-ARIA 란? HTML5 태그만으로 의미를 다 담지 못할 때, 시각장애인 분들이 사용하는 스크린 리더 프로그램에게 "이건 화면을 닫는 버튼이야"라고 친절하게 속삭여주는 고급 기술입니다.

2. 메타(Meta) 태그와 SEO (오픈 그래프)

카카오톡이나 슬랙에 웹사이트 링크를 공유했을 때 뜨는 예쁜 이미지와 제목, 요약글은 전부 <head> 안의 오픈 그래프(Open Graph) 메타 태그 덕분입니다.

<head> <!-- 기본적인 검색엔진 최적화(SEO) --> <meta name="description" content="개발 동아리를 위한 HTML 심화 가이드입니다."> <meta name="keywords" content="HTML, 웹개발, 프론트엔드"> <!-- 검색엔진 봇의 접근 허용 --> <meta name="robots" content="index, follow"> <!-- Open Graph (카카오톡, 페이스북 공유 시 썸네일) --> <meta property="og:type" content="website"> <meta property="og:title" content="DEV. 동아리 HTML 끝판왕"> <meta property="og:description" content="이거 하나면 HTML 마스터!"> <meta property="og:image" content="https://example.com/thumbnail.png"> <meta property="og:url" content="https://devclub.com"> <!-- 트위터 전용 카드 설정 --> <meta name="twitter:card" content="summary_large_image"> </head>

3. 고급 폼(Form)과 정규식 검증

자바스크립트 한 줄 없이도 HTML5만으로 이메일 형식 검사, 비밀번호 패턴 검사(정규식), 그리고 자동완성 검색창을 만들 수 있습니다.

fieldset, datalist, 정규식(Pattern)

<form> <!-- fieldset: 폼 요소를 그룹화하여 시각적 테두리 제공 --> <fieldset> <legend>고급 회원가입</legend> <label>이메일:</label> <!-- type="email"과 required 속성으로 브라우저가 자동 검증 --> <input type="email" required><br> <label>전화번호:</label> <!-- pattern 속성에 정규표현식을 넣어 형식 강제 (010-xxxx-xxxx) --> <input type="tel" pattern="[0-9]{3}-[0-9]{4}-[0-9]{4}" title="010-1234-5678 형식으로 입력하세요"><br> <label>사용 언어:</label> <!-- datalist를 연결하여 구글 검색창 같은 자동완성 기능 제공 --> <input list="langs" name="lang"> <datalist id="langs"> <option value="HTML"> <option value="CSS"> <option value="JavaScript"> </datalist><br> <button type="submit">제출</button> </fieldset> </form>
고급 회원가입


💡 CSS와 연계한 유효성 검사 CSS에서 input:valid 와 input:invalid 선택자를 사용하면, 정규식 조건에 맞을 때는 초록색 테두리, 틀렸을 때는 빨간색 테두리가 나오도록 JS 없이 설정할 수 있습니다!

4. 성능 튜닝 (Script와 Link 최적화)

웹사이트가 로딩될 때 자바스크립트나 폰트 때문에 화면이 하얗게 멈추는 현상을 방지하는 고급 기술입니다.

<head> <!-- 1. preload: 페이지에 꼭 필요한 폰트/이미지를 브라우저가 최우선으로 다운로드하게 지시 --> <link rel="preload" href="main-font.woff2" as="font" type="font/woff2" crossorigin> <!-- 2. prefetch: 사용자가 나중에 방문할 가능성이 높은 페이지 리소스를 백그라운드에서 미리 받아둠 --> <link rel="prefetch" href="/next-page.html"> <!-- 3. JS 로딩 최적화 --> <!-- async: 다운로드가 끝나면 HTML 파싱을 멈추고 즉시 실행 (의존성 없는 독립 스크립트용) --> <script src="analytics.js" async></script> <!-- defer: HTML 파싱을 멈추지 않고 백그라운드 다운로드 후, HTML이 다 그려지면 순서대로 실행 (가장 권장) --> <script src="main.js" defer></script> </head>

5. 글로벌 & 커스텀 데이터 속성

모든 태그에 붙일 수 있는 마법의 속성들입니다. 화면에는 보이지 않지만, 자바스크립트와 연동할 때 필수적인 스킬입니다.

<!-- 1. data-* 속성: 나만의 커스텀 데이터를 태그에 몰래 숨겨두기 --> <!-- (JS에서 dataset.userId 로 쉽게 꺼내 쓸 수 있음) --> <div id="user" data-user-id="998" data-role="admin">김관리</div> <!-- 2. contenteditable: 사용자가 직접 이 태그의 글씨를 수정할 수 있게 만듦 (노션 같은 에디터 원리) --> <p contenteditable="true">이 글씨를 클릭해서 수정해보세요!</p> <!-- 3. draggable: 드래그 앤 드롭이 가능한 요소로 만듦 --> <img src="item.png" draggable="true"> <!-- 4. hidden: CSS의 display:none 과 같은 역할로 완전히 숨김 --> <div hidden>비밀 데이터</div> <!-- 5. loading="lazy": 이미지가 화면에 보일 때쯤 되어서야 다운로드 시작 (트래픽 절약) --> <img src="heavy-image.jpg" loading="lazy" alt="늦게 로딩되는 사진">

이 글씨를 클릭해서 마음대로 수정해보세요!

6. 멀티미디어 심화 & 보안 (Security)

반응형 이미지 처리, 자막 달기, 그리고 외부 콘텐츠를 안전하게 가져오는 방법입니다.

<!-- 1. <picture>: 브라우저가 지원하는 포맷에 따라 최적의 이미지를 골라서 렌더링 (WebP 지원 시 WebP, 아니면 JPEG) --> <picture> <source srcset="photo.webp" type="image/webp"> <img src="photo.jpg" alt="대체 이미지"> </picture> <!-- 2. <video>의 <track>: 접근성을 위한 자막 파일(.vtt) 연결 --> <video controls> <source src="movie.mp4" type="video/mp4"> <track kind="subtitles" src="subs_ko.vtt" srclang="ko" label="한국어" default> </video> <!-- 3. 외부 링크 보안: target="_blank"로 새 창을 열 때 보안 취약점 방지 --> <a href="https://evil.com" target="_blank" rel="noopener noreferrer">안전한 새창 링크</a> <!-- 4. <iframe> 샌드박스: 유튜브 등 외부 사이트를 퍼올 때, 그 사이트의 JS가 내 사이트를 공격하지 못하게 권한 제한 --> <iframe src="https://example.com" sandbox="allow-scripts allow-same-origin"></iframe>
💡 rel="noopener noreferrer" 가 왜 필수인가요? 새 창으로 열린 외부 사이트의 자바스크립트가 악의적인 코드를 품고 있다면, 원래 있던 내 탭의 주소(window.opener)를 맘대로 피싱 사이트로 바꿔버릴 수 있습니다. 이를 완벽하게 차단해 주는 것이 noopener 속성입니다.
