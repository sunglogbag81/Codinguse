1. 실행 컨텍스트와 호이스팅 (Hoisting)

자바스크립트 엔진은 코드를 위에서 아래로 한 줄씩 실행하기 전에, 변수와 함수 선언문을 먼저 메모리에 끌어올려(Hoisting) 기억해 둡니다.

var, let, const의 차이점

var는 호이스팅 시 undefined로 초기화되지만, let과 const는 호이스팅은 되지만 초기화되지 않아 TDZ(일시적 사각지대)에 빠집니다.

// 1. var 호이스팅 (에러 안 남, undefined 출력) console.log(a); var a = 10; // 2. let 호이스팅 (ReferenceError 발생!) // console.log(b); let b = 20; // 3. 함수 선언문 호이스팅 (호출이 먼저 와도 실행됨) hello(); function hello() { console.log("안녕!"); }
> Console Output undefined
안녕!

2. 혼돈의 카오스, 'this' 바인딩

다른 언어에서 this는 객체 자신을 가리키지만, 자바스크립트에서 this는 "함수를 누가 호출했느냐(호출 방식)"에 따라 동적으로 결정됩니다.

일반 함수 vs 화살표 함수

일반 함수의 this는 호출한 객체를 가리키지만, 화살표 함수(Arrow Function)의 this는 자신이 선언된 위치의 상위 환경(Lexical)을 가리킵니다.

const user = { name: "김코딩", // 일반 함수: 자신을 호출한 객체(user)가 this가 됨 sayHi: function() { console.log(this.name); // "김코딩" 정상 출력 }, // 화살표 함수: 호출한 객체를 무시하고, 전역(window) 객체를 가리킴 sayHiArrow: () => { console.log(this.name); // undefined 출력! (window.name을 찾기 때문) } }; user.sayHi(); user.sayHiArrow();
💡 리액트에서 화살표 함수를 쓰는 이유 이벤트 리스너(onClick 등)에서 일반 함수를 쓰면 this가 클릭된 버튼 자체로 바뀌어버려 컴포넌트의 데이터를 못 찾게 됩니다. this를 컴포넌트에 고정시키기 위해 화살표 함수를 씁니다.

3. 클로저 (Closure)

면접 단골 질문 1순위. 클로저는 "함수가 속한 렉시컬 스코프(자신이 선언된 환경)를 기억하여, 함수가 스코프 밖에서 실행될 때도 그 환경에 접근할 수 있는 현상"입니다.

은닉화(Private) 구현하기

외부에서 변수에 직접 접근하지 못하게 막고, 오직 내가 만든 함수를 통해서만 변수를 조작하게 할 때 클로저를 사용합니다. (리액트의 useState 원리와 같습니다.)

function makeCounter() { // 이 변수는 밖에서 절대 직접 수정할 수 없습니다 (은닉) let count = 0; // 내부에 있는 함수(클로저)를 통째로 밖으로 던져줍니다. return function() { count++; return count; }; } const myCounter = makeCounter(); console.log(myCounter()); // 1 console.log(myCounter()); // 2 // console.log(count); // ❌ 에러: count is not defined
> 클로저가 기억한 count 값 1

4. 비동기와 이벤트 루프 (Event Loop)

자바스크립트는 한 번에 하나의 일밖에 못 하는 싱글 스레드(Single Thread)입니다. 그런데 어떻게 여러 통신을 동시에 처리할까요? 오래 걸리는 일(타이머, 네트워크 통신)을 브라우저(Web API)에게 던져놓고 다른 일을 먼저 하기 때문입니다.

Promise와 async/await

콜백 지옥(Callback Hell)을 해결하기 위해 나온 Promise, 그리고 이를 동기 코드처럼 편하게 쓰게 해주는 async / await입니다.

// 서버에서 데이터를 가져오는 가상의 함수 function fetchData() { return new Promise((resolve) => { setTimeout(() => resolve("유저 데이터 수신 완료!"), 2000); }); } // async를 붙이면 이 함수 안에서 await를 쓸 수 있습니다. async function getUser() { console.log("요청을 보냅니다..."); // await: 서버에서 데이터가 올 때까지 '여기서 기다려라!' const result = await fetchData(); console.log(result); // 2초 뒤에 실행됨 } getUser();
💡 이벤트 루프의 실행 순서 Call Stack(현재 실행 중인 코드)이 텅 비어있어야만, Task Queue(대기실)에 있던 비동기 작업 결과들이 Stack으로 올라와 실행될 수 있습니다.

5. 모던 ES6+ 필수 문법 모음

리액트(React)를 하기 위해 반드시 숨쉬듯 자연스럽게 쓸 줄 알아야 하는 현대 문법들입니다.

// 1. 구조 분해 할당 (Destructuring) const user = { name: "길동", age: 25 }; // 객체 안의 값을 꺼내서 바로 변수로 만듭니다. const { name, age } = user; // 2. 전개 구문 (Spread Operator `...`) const arr1 = [1, 2]; const arr2 = [...arr1, 3, 4]; // [1, 2, 3, 4] (배열 껍데기를 벗겨서 뿌림) // 리액트에서 객체 복사할 때 필수! const updatedUser = { ...user, age: 26 }; // 3. 옵셔널 체이닝 (Optional Chaining `?.`) const data = { profile: null }; // data.profile.avatar 를 그냥 찾으면 Null 에러가 나면서 앱이 뻗음. // ?. 를 쓰면 값이 없으면 에러 대신 undefined를 뱉고 안전하게 멈춤. const img = data.profile?.avatar;
